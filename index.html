üåü What is GoodWord.ai?

GoodWord.ai is a specialized web application designed to help Christians find personal guidance and purpose by connecting their life questions to scripture. It uses AI to generate personalized prompts for reflection and offers an interactive chat experience for deeper study, aiming to be a digital companion for faith journeys.

üìö The Problem It Solves

‚ñ∂

The Challenge

Many individuals seek to apply biblical wisdom to their lives but can find it challenging to:

  • Connect broad scriptural themes to specific, personal situations (like a career change or anxiety).
  • Know where to start reading the Bible for relevant guidance.
  • Feel a direct, personal connection to ancient texts.

The GoodWord.ai Solution

GoodWord.ai addresses this by acting as an intelligent bridge. It uses a multi-step personalization process to:

  1. Capture User Context: Gathers information about the user's background, gifts, and current life questions through a guided onboarding.
  2. Generate Hyper-Personalized Prompts: Creates detailed prompts that users can take to their preferred AI, framing their personal situation in a theological context.
  3. Offer a Guided Experience (Pro): Provides an integrated AI "Guide" that interprets scripture, answers questions, and maintains the context of the conversation.
üí° Key Insight: The application's core value is not just providing information, but creating a personalized framework for scriptural reflection that feels relevant and accessible to the modern user.

üèóÔ∏è Technical Architecture

üíª Technology Stack

‚ñ∂

Frontend Technologies

  • HTML5 & CSS3: Standard markup and styling.
  • Tailwind CSS: A utility-first CSS framework for rapid UI development.
  • Vanilla JavaScript: All client-side logic is handled with plain JavaScript, with no external frameworks.
  • Local Storage: Used to persist the user's profile and Pro status across sessions.

AI Integration (Google Gemini)

  • Model for Prompt Curation: `gemini-2.5-flash-preview-05-20` is used to create the initial study prompt in the Pro version.
  • Model for AI Guide Chat: `gemini-2.5-flash-preview-05-20` powers the interactive chat, acting as the theological guide.
  • Model for Text-to-Speech: `gemini-2.5-flash-preview-tts` is used for the audio devotional feature.
‚ö†Ô∏è Critical Security Flaw: The Google Gemini API key is currently stored directly in the frontend JavaScript file (`GoodWordAi.html`). This is a major security risk, as anyone can view the source code and steal the key. This must be rectified before any production deployment.

üöÄ Suggestions for Improvement

üõ°Ô∏è 1. Critical: Secure the API Key

‚ñ∂

The most urgent issue is the exposed API key. This must be fixed by creating a backend.

Recommended Solution:

  1. Create a simple server-side application (e.g., using Node.js/Express, Python/Flask, or Cloud Functions).
  2. Store the API key securely as an environment variable on the server.
  3. Create API endpoints on your server that the frontend can call (e.g., `/api/get-guidance`).
  4. The server endpoint receives the user's prompt, makes the call to the Gemini API using the secure key, and then relays the response back to the frontend.
// Example Node.js/Express backend endpoint app.post('/api/get-guidance', async (req, res) => { const userPrompt = req.body.prompt; const GEMINI_API_KEY = process.env.GEMINI_API_KEY; // Securely loaded // Logic to call the actual Gemini API from the server const response = await callGeminiAPI(userPrompt, GEMINI_API_KEY); res.json(response); });

üíæ 2. Enhance Data Persistence

‚ñ∂

Currently, `localStorage` ties the user's profile and chat history to a single browser. This prevents a seamless cross-device experience.

Recommended Solution:

  • Implement User Authentication: Add a simple sign-in system (e.g., using Firebase Authentication, Auth0, or a simple email/password system).
  • Use a Database: Store user profiles and chat histories in a database like Firestore, PostgreSQL, or MongoDB. This allows users to log in on any device and retrieve their data.
  • Benefit: This also enables features like "Saved Chats" or "My Reflection Journal."

‚ú® 3. Expand Features

  • Sermon/Bible Study Helper: A tool for pastors or small group leaders to generate discussion questions or sermon outlines on a specific passage.
  • Enhanced Fellowship Hall: Add features like replies, upvotes, and user profiles to make the community section more interactive.
